## v1.2.1 — 2026-05-07

### Исправления

**`src/components/CaseOpener.jsx`**
- Исправлен баг: на мобильных разрешениях при замедлении анимации роллера показывалась пустота вместо карточек заданий. Причина — расчёт `finalTranslate` использовал захардкоженную константу `ITEM_WIDTH = 200`, тогда как CSS медиазапросы уменьшают карточки до 150px (≤480px) и 175px (481–768px). Полоса прокручивалась на ~2000px дальше реального контента. Фикс: реальная ширина карточки теперь измеряется из DOM через `firstItem.offsetWidth`, что автоматически учитывает любые CSS-брейкпоинты.

## v1.2.0 — 2026-05-07

### Адаптивная вёрстка (mobile responsive)

#### Изменено

**`src/styles/App.css`**
- `min-height: 100vh` → `min-height: 100dvh` — корректный учёт адресной строки мобильного браузера
- Добавлен `-webkit-text-size-adjust: 100%; text-size-adjust: 100%` — отключает авто-инфляцию шрифтов на iOS/Android
- `@media (max-width: 480px)`: уменьшены внешние отступы контента с `40px 20px` до `20px 16px`

**`src/styles/CaseDisplay.css`**
- Кейс-бокс: `width: 320px; height: 220px` (фиксированные) → `width: min(280px, 88vw); height: auto; aspect-ratio: 320/220` при `max-width: 480px` — корректно масштабируется на любом экране
- Эмблема `∑`: `80px` → `60px` на мобильном
- Заголовок `.case-title__label`: добавлен `text-align: center`; на мобильном — `font-size: clamp(11px, 3.6vw, 22px)` с `white-space: nowrap` и `letter-spacing: 0.03em`
- Подзаголовок: `13px` → `11px` на мобильном
- Дополнительное переопределение `@media (max-width: 360px)`: `font-size: 13px; letter-spacing: 0.04em` (для iPhone SE и ýже)
- `@media (min-width: 481px) and (max-width: 768px)` (планшет): кейс `min(300px, 80vw)`, заголовок `24px`
- Скорректированы внутренние отступы и позиции декоративных полос `.case__stripe`

**`src/styles/CaseOpener.css`**
- `@media (max-width: 480px)`: высота ленты роллера `160px → 120px`; элементы `200×140px → 150×95px`; боковые градиентные маски `120px → 60px`; шрифт заголовка `14px → 12px`
- `@media (min-width: 481px) and (max-width: 768px)`: промежуточные размеры `175×118px`, высота ленты `140px`

**`src/styles/TaskResult.css`**
- `@media (max-width: 480px)`: gap контейнера `28px → 18px`; padding блока сообщения `18px 24px → 12px 16px`; `font-size` текста `16px → 14px`; кнопка «Открыть снова» `12px 32px → 12px 20px`
- `@media (min-width: 481px) and (max-width: 768px)`: промежуточные padding

## v1.1.0 — 2026-05-07

### Реальные задания и нормализация изображений

#### Изменено

**Контент**
- SVG-заглушки заменены на реальные скриншоты заданий ЕГЭ в формате PNG (7 заданий: параллелограмм, вписанный угол, вероятность наушников, вероятность лампочек, логарифмное уравнение, корни, векторы)
- Мотивирующие тексты в `tasks.json` обновлены под содержание каждого конкретного задания с подсказкой по методу решения

**Нормализация изображений**
- Добавлен скрипт `scripts/normalize-tasks.js` (ESM, использует `sharp`): приводит все PNG в `public/tasks/` к единому холсту 600×300 (белый фон, контент центрирован с отступом 24px)
- Скрипт идемпотентен — повторный запуск пропускает уже нормализованные файлы
- Добавлен npm-скрипт `pnpm run normalize` для ручного запуска при добавлении новых заданий
- `prebuild` запускает нормализацию автоматически перед каждой сборкой (в том числе при деплое через GitHub Actions)
- Роллер обновлён: `object-fit: contain` вместо `cover` — задания отображаются целиком без обрезки

**Исправления**
- Исправлен баг React StrictMode в `CaseOpener`: таймер завершения анимации сбрасывался при двойном вызове `useEffect` — экран результата не появлялся
- Исправлена кодировка SVG-заглушек (добавлен XML-заголовок + XML entities для кириллицы)

#### Добавлено
- Cursor Rule `.cursor/rules/normalize-tasks.mdc`: команда «нормализуй картинки» / «добавил новые задания» запускает скрипт нормализации прямо из чата

#### Технический стек (добавлено)
- `sharp 0.34` — обработка изображений в prebuild-скрипте

## v1.0.0 — 2026-05-07

### Первый релиз

#### Реализовано

**Интерфейс**
- Тёмный фон с анимированной сеткой и виньеткой
- Хайтек-кейс по центру экрана: тёмный металл, неоновые циановые углы, пульсирующее свечение, анимированная линия сканирования, эмблема `∑` в шестиугольнике
- Hover-эффект на кейсе: подъём и усиление свечения
- Шрифты: Rajdhani (заголовки / интерфейс) + Inter (текст)

**Механика открытия**
- По клику на кейс запускается анимация роллера в стиле CS2
- Горизонтальная полоса из 50 карточек заданий
- Анимация: `translateX` с `cubic-bezier(0.05, 0.8, 0.15, 1)`, длительность 7 секунд
- Победитель фиксируется на позиции 40 из 50, остановка точно под центральным маркером
- Центральный маркер: вертикальная линия с неоновым свечением и стрелками сверху и снизу
- Боковые маски на роллере (градиентное затухание к краям)

**Экран результата**
- Полноразмерная карточка задания с неоновой рамкой и анимацией появления
- Мотивирующий текст от преподавателя под заданием
- Кнопка «Открыть снова» с анимацией вращения иконки при наведении
- При повторном открытии гарантированно выпадает другое задание (не то же, что в прошлый раз)

**Контент (заглушки)**
- 7 заданий ЕГЭ базового уровня в формате SVG: степени, квадратное уравнение, среднее арифметическое, логарифмы, теорема косинусов, производная, финансовая математика
- Конфигурационный файл `public/tasks/tasks.json` — преподаватель добавляет задания без правки кода

**Деплой**
- Сборка Vite с `base: '/math_case/'` для корректной работы на GitHub Pages
- GitHub Actions workflow `.github/workflows/deploy.yml`: автоматический деплой в ветку `gh-pages` при пуше в `main`

#### Технический стек
- React 18 + Vite 5
- Plain CSS (без UI-фреймворков)
- pnpm 10

#### Известные ограничения v1.0.0
- Задания — SVG-заглушки, не реальные задания с sdamgia.ru (нужна ручная замена)
- Нет интерфейса для загрузки заданий преподавателем (добавляются вручную в `public/tasks/`)
- Только одна «редкость» — все задания равновероятны
